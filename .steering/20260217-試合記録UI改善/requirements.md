# 要求定義書: 試合記録UI改善

## 目的

試合記録画面を中心に、入力安全性・アクセシビリティ・視認性を改善し、誤操作を減らしながら操作の一貫性を高める。

## 背景

- 守備位置の選択値で空文字が `0` に変換され得る実装があり、入力バリデーションをすり抜けるリスクがある。
- 打順入力・試合進行表示・交代モーダルにアクセシビリティ上の欠落がある。
- フォント・コントラスト・タップ領域にばらつきがあり、視認性と操作性にムラがある。

## 要求

### 1. 入力安全性
- 守備位置の未選択値を `''` のまま扱い、`0` にならないこと
- 既存の送信可否判定とバリデーションが維持されること

### 2. アクセシビリティ
- ラインナップ入力欄に行文脈を含むラベル（`id`/`aria-label`）を付与
- 試合ヘッダーの OUT/走者情報にスクリーンリーダー向け要約を付与
- 交代モーダルで Escape 閉じ・初期フォーカス・フォーカストラップを実装

### 3. 視覚整理
- グローバルフォントを Geist 系変数ベースに統一
- 可読性が低い `text-zinc-400` の使用箇所を見直し
- 小型リンクのタップ領域を `min-h-[44px]` 以上に統一

## 受け入れ条件

- [ ] 守備位置の空選択で `0` が保存されない
- [ ] キーボードのみで交代モーダルの開閉と要素移動が完結する
- [ ] スクリーンリーダーで回・表裏・アウト数・走者状態が読み取れる
- [ ] `npm run type-check` が成功する
- [ ] `npm run lint` が成功する
- [ ] 関連テストが成功する
