# 要件: Supabase同期失敗時のエラー通知

## 背景

`syncToSupabase` の呼び出し箇所で `.catch(() => {})` によりエラーが完全に無視されており、
同期失敗時にユーザーへの通知が一切行われない。

ネットワーク断・認証トークン失効・Supabase サーバー障害などが発生した場合でも、
アプリは正常終了したように振る舞う。ユーザーはデータがクラウドに保存されていないことに
気づかないまま試合を終える可能性がある。

## 要件

### 機能要件

1. 同期失敗時に非破壊的な通知を表示する
   - メッセージ: 「クラウド同期に失敗しました。データはローカルに保存されています。」
   - 一定時間後に自動消去（5秒程度）
   - ユーザーが手動で閉じることもできる
2. オフライン時は通知しない（`navigator.onLine` で判定）
3. 開発環境ではコンソールにエラーログを出力する

### 対象呼び出し箇所

- `components/score-input/ScoreInputPage.tsx` — マウント時・試合終了時
- `components/game-setup/GameSetupForm.tsx` — 試合作成後
- `hooks/useOnlineSync.ts` — オンライン復帰時（バッチ同期）

### 非機能要件

- 既存の外部 toast ライブラリは使用しない（プロジェクトに未導入）
- シンプルな自作コンポーネントで実装
- アクセシビリティ: `role="alert"` / `aria-live="polite"` を付与
