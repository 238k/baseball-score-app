# 要件: チーム単位の認証フロー実装

## 背景

GitHub issue #3 に基づく。現状の問題点:
- 個人アカウント前提の UI・テキストになっている
- ローカルストア（Zustand persist）に全ユーザーのゲームが混在し、
  ログインユーザーが異なる場合でも他チームの試合が表示される可能性がある
- ホームページ（`/`）が未認証でもアクセスできる

## 期待する状態

- 認証はチーム単位とする（1チーム = 1 Supabase アカウント、チームメンバーで共用）
- サインアップ時にチーム名を登録する
- ログイン後のホームページにチーム名を表示する
- 試合一覧は自チーム（ログインユーザー）の試合のみ表示する
- 未認証でホームページにアクセスした場合、ログイン画面へリダイレクトする

## 受入条件

- [ ] サインアップフォームにチーム名フィールドがある
- [ ] ホームページにチーム名が表示される
- [ ] ローカルストアに他チームの試合が混在しても、現在のログインユーザーの試合のみ表示される
- [ ] 未認証時は `/` にアクセスすると `/login` にリダイレクトされる
- [ ] ログアウトボタンが存在する

## 対象ファイル

- `app/(auth)/signup/page.tsx` — チーム名フィールド追加
- `app/(auth)/login/page.tsx` — テキストをチームアカウント向けに変更
- `app/page.tsx` — チーム名表示・ゲームフィルタリング・ログアウトボタン
- `middleware.ts` — `/` を認証必須に追加
- `hooks/useAuth.ts` — チーム名を返す
